ADAUGAREA UNUI UTILIZATOR NOU

Pasul 1: Definirea unei vederi pentru creare utilizator

In fisierul views.py din aplicatia userextend se defineste o clasa numita UserCreateView. Aceasta clasa mosteneste CreateView si contine urmatoarele atribute:

template_name – fisierul HTML care va fi afisat

model – modelul utilizatorului

form_class – formularul folosit pentru creare utilizator

success_url – URL-ul catre care utilizatorul va fi redirectionat dupa inregistrare cu succes

Pasul 2: Definirea unei rute URL pentru creare utilizator

In fisierul urls.py din aplicatia userextend se adauga o ruta noua care foloseste clasa UserCreateView. Astfel, se defineste o cale accesibila in browser unde utilizatorii se pot inregistra.

Pasul 3: Adaugarea unui buton "Sign up" in bara de navigare

In fisierul HTML al barei de navigare (navbar.html), se adauga un buton sau link care duce utilizatorii la pagina de inregistrare (creare cont nou). Acesta foloseste calea definita anterior.

Pasul 4: Restrictionarea accesului pentru utilizatorii neautentificati

In aplicatiile employee, manager si intro se adauga LoginRequiredMixin in clasele de vedere pentru a permite accesul doar utilizatorilor autentificati. Pentru functii, se foloseste decoratorul @login_required() cu acelasi scop. Daca utilizatorul nu este autentificat, acesta este redirectionat automat catre pagina de login.



LoginRequiredMixin vs login_required()

LoginRequiredMixin este o clasa folosita in clasele de view (views bazate pe clase) pentru a restrictiona accesul la resurse. Daca utilizatorul nu este autentificat, va fi redirectionat la login.

login_required() este un decorator folosit pentru functii (views bazate pe functii). Are acelasi efect: restrictioneaza accesul daca utilizatorul nu este logat si il redirectioneaza catre pagina de login.

Ambele metode sunt folosite pentru a proteja rutele si continutul aplicatiei si pentru a permite accesul doar utilizatorilor autentificati.
